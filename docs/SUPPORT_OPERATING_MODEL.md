## Support Categories

### Install

- app will not open
- installer fails
- update behavior is unclear
- unsupported platform confusion

### Billing

- checkout failed
- customer paid but app stayed locked
- restore purchase failed
- portal or cancellation issue

### Product

- trusted run failed
- recovery feels broken
- workspace selection confused the user
- output or proof is unclear

### Account and Entitlement

- tier mismatch
- expired or downgraded state confusion
- activation does not survive relaunch

## Response Expectations

- launch blockers: same day
- paid customer access issues: same day
- install failures on supported platforms: same day
- normal product bugs: next business day target
- feature requests: triage weekly

## Standard Intake Fields

Every support issue should capture:

- app version
- platform
- whether customer is paid, trial, or free
- short problem summary
- screenshot if available
- whether the issue blocks install, payment, activation, or usage

If relevant, also capture:

- purchase email
- customer ID
- run ID
- receipt/session reference

## Triage Rule

### Severity 1

- cannot install on supported platform
- paid customer cannot unlock product
- restore purchase broken
- shipping build is broken

### Severity 2

- trusted run confusing or inconsistent
- recovery broken but workaround exists
- upgrade prompt or plan enforcement mismatch

### Severity 3

- cosmetic issue
- copy issue
- non-critical friction

## Support Workflow

1. confirm category and severity
2. confirm whether the issue is reproducible
3. link the issue to the relevant runbook or checklist
4. respond with next step, not just acknowledgment
5. log the root cause after resolution

## Required Support Assets

- live support inbox
- current release/version reference
- restore-purchase instructions
- diagnostic/support bundle instructions
- known supported platforms
- known issues list or internal equivalent

## Weekly Review

Review these once per week:

- top recurring support issue
- top launch blocker from support
- average time to first response
- number of billing and entitlement issues
- number of install failures

The rule is simple:

- if the same issue appears three times, it is now a product problem, not just a support problem
