# GTM Operating Plan

Date: 2026-03-29

This is the repo-safe go-to-market operating plan for RinaWarp Terminal Pro.

It is not a private investor memo and it is not a personal finance document.

## Core Story

RinaWarp Terminal Pro is a proof-first agent workbench for developers who want AI help without losing trust, receipts, or recovery.

## Primary Audience

- indie developers
- technical founders
- small engineering teams
- developers frustrated by opaque AI coding tools

## Primary Message

- ask naturally
- run through one trusted path
- inspect proof when it matters
- recover without losing the thread

## Channel Priority

Start with one strong channel before expanding.

### Priority 1

- product demo clips
- founder posts with real workflows
- launch posts in developer communities

### Priority 2

- documentation and SEO around proof-backed terminal workflows
- customer stories
- comparison pages against generic AI terminals

### Priority 3

- referral loop
- partnerships
- affiliate or community sponsorship tests

## Landing Page Rule

The homepage must answer in under ten seconds:

- what this is
- who it is for
- why it is different
- why it is trustworthy

## Weekly GTM Tasks

- publish one real workflow demo
- review signup and activation numbers
- identify one conversion bottleneck
- tighten one message, screenshot, or onboarding step

## Initial Experiments

- homepage headline variants
- pricing page clarity variants
- longer trial versus shorter trial
- onboarding with quick demo versus standard welcome
- proof-first message versus generic AI-terminal message

## Do Not Do Yet

- too many plans
- too many audiences
- broad enterprise motion before the self-serve path works
- growth channels you cannot measure

## Success Criteria

The GTM system is healthy when:

- acquisition source is visible
- activation rate is measurable
- paid conversion is measurable
- top drop-off point is known
- messaging changes can be tied to behavior changes
