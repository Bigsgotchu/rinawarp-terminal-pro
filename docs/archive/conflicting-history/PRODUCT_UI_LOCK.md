# RinaWarp Terminal Pro Product UI Lock

## Canonical Visual Reference

The canonical visual target is the dark Agent Shell screenshot generated by:

```bash
bash apps/terminal-pro/scripts/run-electron-playwright.sh agent-thread-visual-smoke.spec.ts
```

Reference artifact:

```text
apps/terminal-pro/e2e/screenshots/agent-shell-initial-launch.png
```

That screenshot is the visual direction: premium, dark, focused, technical, compact, trustworthy, and execution-first.

## Vocabulary

- Product: RinaWarp Terminal Pro
- AI copilot: Rina
- Desktop container: Agent Shell
- Primary user experience: Agent Thread
- Secondary detail surface: Inspect rail
- User-facing trust layer: Proof
- Internal export artifact: Receipt

RinaWarp Terminal Pro is not a workbench, chatbot, terminal wrapper, dashboard, code generator, CLI assistant, or button launcher.

## Visual Identity

RinaWarp Terminal Pro should feel like a premium AI copilot shell for real computer work.

It must be:

- dark, polished, and professional
- chat-first through the Agent Thread
- compact and terminal/operator-like
- accented with magenta/pink and teal/cyan
- proof-backed and execution-aware

It must not be:

- white or raw HTML
- bubbly
- dashboard-heavy
- cluttered with competing panels
- positioned as a workbench product

```css
--bg-app: #0b0d10;

--glow-accent: 0 0 32px rgba(255, 47, 179, 0.24), 0 0 22px rgba(38, 247, 212, 0.12);
```

Use four dark background layers:

- app background
- panel background
- raised card/run block background
- hover/active background

No plain white surfaces are allowed in the main app shell.

## Layout Lock

Left rail:

- workspace/account card
- main navigation
- marketplace shortcut
- mode/status area
- profile/footer area
- narrow, useful, and not crowded

Center panel:

- Agent Thread is the visual focus
- messages, execution stream, proof summary, and composer live here
- run blocks belong inside the conversation timeline

Right rail:

- Terminal
- Logs
- Diff
- Proof
- secondary inspect surface only

## Interaction Lock

Keep:

- natural-language composer
- typed conversation
- execution stream in the thread
- short reasoning
- proof attached inline
- inspect side rail
- compact quick suggestions that only fill the composer

Prevent:

- giant bubble cards everywhere
- duplicate panels
- white empty-state pages
- fake placeholder surfaces
- one-click unsafe mutation buttons
- "workbench" as product identity
- redundant navigation that repeats the same destination

## Agent Thread Feel

The thread should feel chat-style, concise, live, terminal-aware, and proof-backed.

A run should read as an execution story:

1. user asks naturally
2. Rina explains what she understands
3. Rina shows a plan
4. command/run block appears inline
5. result and verification appear inline
6. proof attached appears inline

## Typography And Spacing

Typography:

- clean sans-serif UI
- strong hierarchy
- small technical labels
- readable body text
- monospace only for terminal output, code, and command text

Spacing:

- tighter, not airy
- compact controls
- minimal wasted space
- consistent panel padding
- narrow gaps between related items

Corners:

- medium rounded
- not pill-everything
- not big bubbly cards

## Banned Regressions

These must never return:

- white raw HTML shell
- legacy React shell
- React `#root`
- old stale placeholder UI
- "Rina workbench" product wording
- "AI workbench" product wording
- duplicate receipts/proof/recovery surfaces
- giant empty-state action cards
- multiple competing center experiences
